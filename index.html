Modul pelatihan di dalam analisis ini seluruhnya menggunakan Python di dalam notebook Google Colab, dengan pendekatan prompt-driven — di mana GenAI/LLM digunakan sebagai mitra pemrograman.

  • Dataset: TerraClimate — data iklim spatio-temporal global, dengan variabel seperti suhu, curah hujan, kelembaban, dan lainnya
  • Alat bantu: Google Colab + Python + GenAI (ChatGPT, Gemini, atau lainnya)
  • Satuan waktu: Bulanan dan Tahunan

Asumsi kunci: Tren iklim dapat mempengaruhi risiko banjir maupun panas kemarau secara signifikan, terutama melalui variabel suhu dan curah hujan

Overview

Pada akhir sesi ini, peserta akan mampu:

  1. Menggunakan Python di dalam Kaggle atau Google Colab
  2. Membuka dan membaca data iklim ke dalam Google Colab
  3. Mengeksplorasi pola tren suhu dan curah hujan dengan grafik

Apa itu Kaggle?

Kaggle adalah platform kompetisi data science yang menyediakan dataset, notebook, dan lingkungan coding berbasis browser untuk analisis data. Kaggle memungkinkan pengguna untuk:

  1. Menjelajahi dan mengunduh dataset secara gratis
  2. Membuat dan berbagi notebook interaktif dengan komunitas data science global
  3. Berpartisipasi dalam kompetisi data science untuk mengasah keterampilan analisis dan machine learning.

Langkah-langkah menggunakan Kaggle

Apa itu Google Colab?

Google Colab (singkatan dari “Colaboratory”) alat bantu gratis, lingkungan notebook berbasis browser dimana Anda dapat menjalankan kode Python secara online tanpa instalasi rumit di desktop/komputer lokal.

Colab dapat membantu dalam:

  1. Run notebook langsung dari link yang dibagikan
  2. Eksekusi kode dalam cell kecil dan inspeksi output langkah demi langkah
  3. Kombinasi teks naratif, prompt, kode, dan gambar dalam satu tempat
  4. Simpan salinan Anda sendiri di Google Drive untuk latihan berkelanjutan

Apa itu Shiny?

Shiny adalah framework open-source untuk membangun aplikasi web interaktif dengan R atau Python. Shiny memungkinkan pengguna untuk membuat dashboard dan visualisasi data yang dinamis tanpa perlu pengetahuan mendalam tentang pengembangan web.

Dalam proses hosting Shiny dashboard, diperlukan langkah-langkah pembuatan akun di shinyapps.io dan menginstall rsconnect-python. Berikut ini tahapan pembuatan akun:

  1. Kunjungi shinyapps.io dan klik “Sign Up” untuk membuat akun baru.
  2. Isi formulir pendaftaran dengan informasi yang diperlukan, seperti nama, email, dan password.
  3. Setelah mendaftar, pilih nama domain unik untuk Shiny Apps yang diinginkan. Kemudian Save.
  4. Akun siap digunakan dan dikustomisasi untuk hosting aplikasi Shiny.
TipTips untuk first-time users
  1. Gunakan Runtime > Run all untuk eksekusi semua cell secara berurutan
  2. Jika gagal, baca errornya, perbaiki prompt/kodenya, lalu jalankan ulang cell tersebut
  3. Jika runtime terputus, sambungkan kembali dan jalankan dari atas

Skor Kredit Berbasis Risiko Iklim (Banking Relevance)

Bagian ini menghubungkan data iklim dengan praktik perbankan sesuai framework TCFD (Task Force on Climate-related Financial Disclosures) dan regulasi OJK SEOJK No. 16/2021 tentang Keuangan Berkelanjutan.

Mengapa bank perlu climate risk scoring?

  • Physical risk (banjir, kekeringan, suhu ekstrem) → NPL naik di daerah terdampak
  • Transition risk (carbon tax, regulasi hijau) → stranded assets sektor fosil
  • Nilai agunan properti turun akibat sea-level rise atau bencana berulang

Formula Skor Kredit Dasar (sesuai dashboard)

Prompt (copy/paste)

Persona:
Kamu adalah quant analyst di divisi manajemen risiko bank komersial Indonesia.

Task:
Tulis kode python untuk menghitung climate-adjusted credit score per kabupaten:

Formula:
  credit_score = 700 - 120 * norm(tmax_annual) - 80 * norm(ppt_std) + noise(σ=15)
  diclip ke range [300, 850]

Interpretasi:
- Daerah dengan suhu maksimum tinggi: risiko lebih tinggi (kredit score turun)
- Daerah dengan variabilitas curah hujan tinggi: risiko lebih tinggi (kredit score turun)

Langkah:
1. Normalisasi tmax_annual dan ppt_std menggunakan MinMaxScaler ke [0, 1]
2. Hitung credit_score menggunakan formula di atas dengan random seed = 42
3. Tambahkan kolom risk_category: Very High (<550), High (550-599),
   Medium (600-649), Good (650-699), Very Good (≥700)
4. Plot distribusi histogram credit score seluruh kabupaten
5. Plot top 10 kabupaten skor tertinggi dan terendah
6. Plot scatter plot tmax_annual vs credit_score dengan warna risk_category

Input: df_kab dari Section 4
Output: df_kab dengan kolom tambahan credit_score dan risk_category

Constraints:
- Gunakan sklearn.preprocessing.MinMaxScaler
- Set random seed = 42
- Buat visualisasi menarik dengan seaborn dan matplotlib

Analisis Portfolio Kredit Bank

Prompt (copy/paste)

Persona:
Kamu adalah analis risiko kredit di bank komersial Indonesia yang menerapkan
framework TCFD untuk stress-testing portofolio kredit.

Task:
Tulis kode python untuk mensimulasikan portofolio kredit bank dan mengukur
eksposur risiko iklim:

1. Generate data sintetis portofolio kredit (gunakan random seed=42):
   - Merge df_kab (dari Section 5.1) dengan simulasi pinjaman per kabupaten
   - Tambahkan kolom: sector (Agriculture/Manufacturing/Mining/Real Estate/Retail),
     outstanding_billion_idr (50–2000 miliar per kabupaten), npl_base (1-5%)
   - Sector climate multiplier: Agriculture=1.8, Real Estate=1.5, Mining=1.3,
     Manufacturing=1.1, Retail=1.0

2. Hitung Physical Risk Score dan ECL (Expected Credit Loss) add-on:
   - physical_risk = climate_risk_index * sector_multiplier
   - npl_adjusted = npl_base * (1 + physical_risk)
   - ecl_base = outstanding * npl_base * 0.45  # LGD = 45%
   - ecl_climate_addon = outstanding * (npl_adjusted - npl_base) * 0.45

3. Hitung Loan-at-Risk (LaR):
   - LaR = total outstanding dimana climate_risk_index > 0.4
   - Hitung sebagai % dari total portfolio

4. Buat visualisasi:
   - Bar chart ECL add-on per sektor
   - Scatter plot LaR vs rata-rata credit score per provinsi
   - Tabel ringkasan risiko 10 provinsi tertinggi

5. Print key metrics:
   - Total ECL add-on (climate)
   - Loan-at-Risk (IDR dan %)
   - Provinsi dengan risiko tertinggi

Constraints:
- Gunakan pandas, numpy, matplotlib, seaborn
- Simpan ke df_portfolio (seluruh data) dan df_summary (ringkasan per provinsi)
- Format angka dalam miliar IDR

Analisis Skenario NGFS (Network for Greening the Financial System)

Prompt (copy/paste)

Persona:
Kamu adalah analis senior climate stress-testing di bank, menggunakan skenario
NGFS (Network for Greening the Financial System) untuk menilai dampak iklim
ke portofolio kredit.

Context:
NGFS mendefinisikan 4 skenario utama:
1. Net Zero 2050: Transisi cepat + suhu naik 1.5°C
   (physical_risk_mult=0.8, transition_risk_mult=1.5, carbon_price_2030=75)
2. Delayed Transition: Terlambat beradaptasi + suhu naik 1.8°C
   (physical_risk_mult=0.9, transition_risk_mult=2.0, carbon_price_2030=100)
3. Current Policies: Kebijakan saat ini + suhu naik 3.0°C
   (physical_risk_mult=1.8, transition_risk_mult=0.8, carbon_price_2030=10)
4. Hot House World: Tanpa aksi + suhu naik 4.0°C
   (physical_risk_mult=2.5, transition_risk_mult=0.5, carbon_price_2030=5)

Task:
1. Untuk setiap skenario NGFS, hitung projected ECL dengan:
   projected_ecl = baseline_ecl * (physical_risk_mult * 0.6 + transition_risk_mult * 0.4)
2. Hitung ECL increase (%) vs baseline
3. Untuk sektor karbon-intensif (Mining, Energy, Manufacturing), terapkan
   tambahan transition_risk_penalty = carbon_price_2030 * 0.001 * outstanding
4. Buat visualisasi:
   - Bar chart perbandingan total ECL per skenario
   - Heatmap: skenario vs sektor (ECL increase %)
   - Line chart: kenaikan suhu vs ECL impact
5. Buat tabel ringkasan executive summary untuk manajemen bank

Constraints:
- Gunakan pandas, numpy, matplotlib, seaborn
- Format tabel dengan angka jelas (miliar IDR, persentase)
- Tambahkan interpretasi singkat setiap skenario
- Relevansi regulasi: sebutkan kaitannya dengan OJK SEOJK No.16/2021
